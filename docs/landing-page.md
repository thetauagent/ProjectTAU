# ProjectTAU Landing Page

## Purpose
The landing page is the first screen in the web shell. It should feel calm, futuristic, and immediately useful while still using dummy content.

## Goals
- make the product identity obvious in seconds
- provide quick entry into the dashboard and project builder
- show the current system state at a glance
- keep the page simple, spacious, and easy to scan
- leave room for the Godot companion layer as a separate experience

## Content blocks
1. **Top bar**
   - ProjectTAU logo/title
   - simple status pill
   - primary action button

2. **Hero section**
   - short headline
   - one-sentence explanation
   - primary and secondary actions

3. **Quick actions**
   - Open dashboard
   - Start project builder
   - Review workspace
   - Launch companion layer

4. **Workspace overview**
   - active modules
   - placeholder cards for dashboard areas
   - recent activity summary

5. **System status**
   - TAU online
   - voice idle
   - sync healthy
   - Godot layer available

## Visual direction
- dark, soft, high-contrast surface
- subtle orange gradients and glows
- rounded cards
- clear typographic hierarchy
- restrained motion rather than heavy animation
- slightly more futuristic scanline / glass treatment

## Layout rule
The landing page should read like a control room, not a marketing site.
It should invite action first and explanation second.

## Content rules
- use dummy content only
- no real project data yet
- no dense paragraphs
- no cluttered dashboard chrome on the landing page
